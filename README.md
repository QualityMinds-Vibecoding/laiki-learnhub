# LAiKi LearnHub

LearnHub is the shared, company-visible side of LAiKi.

LAiKi is a personal, local-first learning agent that runs on each consultant's
machine. When a consultant approves a learning resource, that resource (and
nothing personal about them) can be contributed here.

## How contributions arrive

1. A LAiKi user approves a resource during their session.
2. At the start of the next session, LAiKi detects unsynced contributions and
   asks once: "You have N resources not yet shared to LearnHub — want me to
   push them?"
3. The user says yes → `sync-to-learnhub.sh` (in the `laiki/` repo) writes a
   contributor file to `contributions/` and commits it here.
4. A maintainer (or CI) runs `node scripts/merge.js` to regenerate
   `aggregated/learnhub.json`.
5. A dashboard reads `aggregated/learnhub.json` to display topics, head
   counts, and resources.

## Contribution format

Each file in `contributions/` is a JSON array. Each entry:

```json
{
  "topic": "terraform",
  "resource": {
    "url": "https://...",
    "title": "...",
    "type": "article|video|podcast|course|tool|book",
    "language": "en|de|..."
  },
  "topic_token": "<random-hex>",
  "approved_at": "2026-06-25"
}
```

### Privacy design

`topic_token` is an anonymous, per-topic identifier generated randomly on the
contributor's machine. It is **not** derived from any identity. Two entries
from the same person on the same topic share a token (so head counts are
accurate). Two entries from the same person on different topics have entirely
unrelated tokens — cross-topic profiling is not possible.

No personal data is ever written here. No names, no project references, no
reasons a resource was added. The privacy boundary is enforced by LAiKi.

## Directory structure

```
contributions/     ← one JSON array file per LAiKi user
topics/
└── topics.json    ← canonical topic list (synced from LAiKi)
aggregated/
└── learnhub.json  ← merged aggregate (generated by merge script)
scripts/
└── merge.js       ← aggregation script
```

## Generating the aggregate

```bash
node scripts/merge.js
```

Reads all `contributions/*.json`, computes per-topic head counts (distinct
tokens active in the last 180 days), aggregates resources, and writes
`aggregated/learnhub.json`. Monthly timeline data stores counts only — raw
tokens are discarded after aggregation.

## Dashboard setup

Point your dashboard at `aggregated/learnhub.json`. Structure:

```json
{
  "generated_at": "2026-07-07T...",
  "topics": [
    {
      "id": "terraform",
      "name": "Terraform",
      "head_count": 14,
      "resource_count": 38,
      "timeline": [{ "month": "2026-04", "heads": 5 }, ...],
      "resources": [{ "url": "...", "approval_count": 23, ... }]
    }
  ]
}
```
